## 1、为什么要用快捷键

在使用 Jira 管理需求的日常工作中，大量时间花在创建 Issue、编辑字段、切换视图、搜索过滤上。如果每次操作都要用鼠标点来点去，效率会大打折扣。

Jira 内置了丰富的快捷键，熟练使用后可以大幅提升操作效率。这篇文章整理了日常使用中最实用的快捷键，按使用场景分类介绍。

> **提示**：首次使用时，可以按 `?` 查看当前页面的所有可用快捷键。不同页面快捷键可能略有差异。

## 2、全局快捷键

全局快捷键在任何页面都可以使用：

| 快捷键 | 功能 |
|--------|------|
| `g` + `d` | 跳转到 Dashboard（仪表盘） |
| `g` + `p` | 跳转到我的项目列表 |
| `g` + `b` | 跳转到当前项目的 Backlog（待办列表） |
| `g` + `c` | 跳转到当前项目的看板（Board） |
| `g` + `s` | 跳转到当前项目的 Sprint（迭代列表） |
| `/` | 聚焦到搜索框 |
| `?` | 打开快捷键帮助面板 |
| `j` | 选中下一个 Issue |
| `k` | 选中上一个 Issue |
| `o` 或 `Enter` | 打开当前选中的 Issue |
| `e` | 快速编辑当前 Issue |
| `m` | 添加评论 |
| `s` | 关注/取消关注当前 Issue |
| `l` | 聚焦到经办人字段 |
| `a` | 聚焦到经办人字段并打开人员选择器 |
| `u` | 打开当前 Issue 的链接面板 |
| `t` | 切换 Issue 的状态 |
| `Tab` | 在当前 Issue 详情页的各字段间切换 |
| `Esc` | 关闭当前弹窗或取消选择 |

> **`g` 组合键规律**：`g` 是跳转（Go）的缩写，之后按 d/p/b/c/s 分别对应 Dashboard/Projects/Backlog/Board/Sprints，记住这个规律就能记住一串。

## 3、Issue 列表页快捷键

在 Issue 列表（如 Backlog、看板、Sprint 视图中）使用：

| 快捷键 | 功能 |
|--------|------|
| `c` | 创建新 Issue |
| `/` | 打开 JQL 高级搜索 |
| `g` + `i` | 打开"我的 Issue"快速过滤器 |
| `,` | 打开当前项目的设置 |
| `m` | 批量添加评论（选中多个 Issue 后） |
| `b` | 批量分配经办人（选中多个 Issue 后） |
| `p` | 批量移动到项目 |
| `d` | 批量删除（需谨慎） |

### 列表导航

| 快捷键 | 功能 |
|--------|------|
| `j` | 选中下一行 Issue |
| `k` | 选中上一行 Issue |
| `Space` | 展开/折叠选中 Issue 的详情 |
| `→` | 展开选中 Issue 的子任务或关联 |
| `←` | 收起展开的内容 |
| `Home` | 跳到列表开头 |
| `End` | 跳到列表末尾 |

## 4、Issue 详情页快捷键

打开一个 Issue 后，在详情页可以使用：

| 快捷键 | 功能 |
|--------|------|
| `e` | 快速编辑（弹出编辑弹窗） |
| `a` | 修改经办人 |
| `l` | 聚焦到经办人字段 |
| `t` | 修改状态 |
| `p` | 修改优先级 |
| `m` | 添加评论 |
| `s` | 切换关注状态 |
| `o` | 打开原始评论视图 |
| `r` | 回复选中的评论 |
| `y` | 切换显示/隐藏已完成的工作流步骤 |
| `v` | 打开投票面板 |
| `w` | 打开监视者面板 |
| `u` | 打开链接面板 |
| `Esc` | 关闭详情页 |

### 评论编辑器

| 快捷键 | 功能 |
|--------|------|
| `Ctrl + Enter` | 提交评论 |
| `Ctrl + Shift + C` | 清除评论内容 |
| `Ctrl + B` | 加粗 |
| `Ctrl + I` | 斜体 |
| `Ctrl + Z` | 撤销 |
| `Ctrl + Shift + Z` | 重做 |

## 5、编辑器中的快捷操作

在 Issue 描述、评论等编辑器中：

| 快捷键 | 功能 |
|--------|------|
| `@` | 提及用户（输入 `@` 后会弹出用户选择器） |
| `#` | 引用另一个 Issue（输入 `#` 后会弹出 Issue 选择器） |
| `{` | 插入宏（如 `{code}...{code}`） |
| `!` | 插入附件占位符 |

例如，输入 `[PROJ-123]` 会自动解析为可点击的 Issue 链接。输入 `/quote` 可以快速插入引用块。

## 6、看板视图快捷键

在 Scrum 或 Kanban 看板中：

| 快捷键 | 功能 |
|--------|------|
| `c` | 在当前泳道创建 Issue |
| `s` | 打开当前泳道设置 |
| `n` | 创建新泳道 |
| `r` | 重命名泳道 |
| `d` | 删除泳道 |
| `,` | 打开看板设置 |

### 泳道内操作

| 快捷键 | 功能 |
|--------|------|
| `j` | 选中下一张卡片 |
| `k` | 选中上一张卡片 |
| `→` / `←` | 将卡片移动到相邻列 |
| `↑` / `↓` | 在列内上下移动卡片 |
| `Space` | 展开/折叠卡片详情 |
| `o` | 打开卡片详情 |

## 7、搜索与过滤快捷键

| 快捷键 | 功能 |
|--------|------|
| `/` | 快速搜索，跳到搜索框 |
| `g` + `f` | 打开过滤器管理 |
| `Ctrl + /` | 切换快速过滤（Quick Filter） |
| `Ctrl + S` | 保存当前搜索为过滤器 |

### 快速过滤器

在看板顶部通常有快速过滤栏，可以用 `Tab` 聚焦后输入关键词过滤卡片。

## 8、浏览器通用快捷键

除了 Jira 自身的快捷键，浏览器中也有一些常用操作：

| 快捷键 | 功能 |
|--------|------|
| `Ctrl + L` | 聚焦地址栏 |
| `Ctrl + R` | 刷新页面 |
| `Ctrl + D` | 收藏当前页面（加入书签） |
| `Ctrl + Tab` | 切换到下一个标签页 |
| `Ctrl + Shift + Tab` | 切换到上一个标签页 |
| `Ctrl + T` | 新建标签页 |
| `Ctrl + W` | 关闭当前标签页 |
| `Ctrl + F` | 在当前页面查找 |

## 9、自定义快捷键

Jira 支持管理员为项目自定义部分快捷键：

- **工作流转换快捷键**：为常用的状态转换设置键盘快捷键
- **批量操作快捷键**：为批量分配、批量移动等设置快捷键

设置路径：**Project Settings → Shortcuts**

## 10、实践建议

几点使用快捷键的实践建议：

1. **从高频操作开始**：先记住 `j`/`k`（上下移动）、`o`（打开）、`e`（编辑）、`m`（评论），这四个覆盖了 80% 的日常操作。
2. **善用 `?` 帮助**：遇到不确定的快捷键时，按 `?` 查看当前页面的完整快捷键列表。
3. **记住 `g` 组合键**：`g` 是跳转（Go）的缩写，之后按 d/p/b/c/s 分别对应 Dashboard/Projects/Backlog/Board/Sprints。
4. **不要硬背，用中自然记住**：频繁使用的操作重复几次后，手指会自然记住快捷键，不需要刻意背诵。

## 11、总结

快捷键是提升效率的"杠杆"，投入少量时间学习，就能长期节省大量鼠标操作时间。建议从这四个最常用的开始：`j`/`k` 导航、`o` 打开、`e` 编辑、`m` 评论，熟练后再逐步扩展到其他场景。

每天前进一小步，就是一个新的高度！
